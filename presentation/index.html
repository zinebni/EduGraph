⚙️ Workflow

LangGraph Multi-Agent Workflow

Every generation follows a strict, deterministic pipeline — the supervisor enforces sequential agent execution, ensuring no step is skipped.

📋
Step 1
User Submits Request
User enters a topic, target audience, skill level (Beginner→Expert), and total duration. Optionally adds custom focus instructions.
🔌
Step 2
WebSocket Connection Opens
FastAPI opens a WebSocket channel. The LangGraph workflow is compiled dynamically with the user's API keys and model selection.
🔍
Step 3 — search_agent
Market Research
Performs ≥3 Tavily searches: job market demands, modern tool/tech stack, educational best practices. Compiles a structured research report.
✍️
Step 4 — curriculum_writer_agent
Curriculum Authoring
Reads the full research report from shared MessagesState and writes a comprehensive Markdown curriculum: 5–8 modules, detailed lessons, timelines, objectives.
📝
Step 5 — quiz_exercise_agent
Assessment Generation
Reads the curriculum and generates structured JSON: MCQ questions with answer explanations + progressive coding exercises per module.
📦
Step 6
Output Delivered
Supervisor compiles all agent outputs. Results saved to SQLite. Frontend receives the final package and renders the interactive accordion UI.
🔁 Shared State — MessagesState

All agents coordinate via a single, chronologically-ordered list of AIMessage objects. Each agent appends its output — subsequent agents automatically see all prior context.

📋 User message → MessagesState
🔍 Research AIMessage appended
✍️ Curriculum AIMessage appended
📝 Quizzes JSON AIMessage appended
📦 Final compiled output → DB + UI
🔒 Supervisor Enforcement Rules
  • → Exact sequence: Search → Write → Assess
  • → Cannot finish before quiz_exercise_agent completes
  • → Full context passed between every agent
  • → Level / audience constraints enforced throughout

🛠️ Engineering

Development Techniques

EduGraph applies cutting-edge AI engineering principles to build a robust, scalable, and maintainable multi-agent system.

01
Multi-Agent Systems
Decomposing a complex task (curriculum generation) into specialized, focused agents — each expert in one domain — dramatically improves output quality over a single monolithic LLM call.
02
Agent Orchestration
LangGraph's create_supervisor() creates a graph-based supervisor that uses structured tool-calling to route between agents, enforcing the exact execution sequence.
03
Shared State Management
All agents operate on a single MessagesState — a chronological list of AIMessages. Each agent reads all prior context before producing its output, enabling implicit inter-agent communication.
04
Graph-based Workflows
LangGraph models the agent pipeline as a compiled directed graph. Nodes are agents; edges are supervisor routing decisions. This enables conditional branching, loops, and deterministic execution.
05
Prompt Engineering
Each agent has a carefully engineered system prompt defining its exact role, constraints, output format, and what it must NOT do — preventing scope creep and hallucinated outputs across agent boundaries.
06
AI Pipeline Design
The three-stage pipeline (Research → Write → Assess) follows a producer-consumer pattern: each stage consumes the previous stage's output, progressively enriching the educational content.
07
Modular Architecture
Clean separation: agents/, routes/, tools/, components/ directories. The graph is built dynamically (build_graph()) at runtime with injected API keys — fully swappable LLM backends.
08
Event-driven Streaming
WebSocket-based event streaming pushes agent lifecycle events (start, result, complete, error) to the frontend in real time — giving users live visibility into the AI pipeline execution.
